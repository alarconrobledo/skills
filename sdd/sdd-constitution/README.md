# sdd-constitution

## Qué hace
Define o edita los principios no negociables del proyecto: arquitectura, testing, seguridad, dependencias y estándares de código.

## Cuándo usarla
- Al inicio del proyecto para establecer las reglas base
- Cuando quieres añadir o modificar principios del proyecto
- Si el equipo necesita alinear estándares técnicos

## Cómo invocarla
```
/sdd:constitution
/sdd:constitution [category]
```

## Genera / modifica
- `constitution.md` — documento de principios del proyecto

## Flujo SDD
Se usa al inicio (tras `sdd:init`) o cuando se necesita revisar/ampliar las reglas del proyecto.

## Origen
Workflow SDD — Specification-Driven Development
